Core

The core of the engine is the basic framework that's needed for all of the other systems to cooperate, such as the event system, as well as the entity and entity component, world cell, resource systems.

It consists of the basic type definitions for an entity, an entity component, a world cell, a resource, events, messages, as well as templates for data storage types.

The code for the core of the engine can be found in the core.h header file. There's also some functions and methods implemented in the core.cpp file.

UID

To save space the engine stores all of the strings (model, material, entity names, etc.) as a one very large string. It also ensures that every substring is stored only once.

You can use the Core::UID() function to store a string. The function will return a name_t (which is an alias for an unsigned 64-bit long integer). It also guarantees that identical strings will all return the same name_t. This significantly reduces time when comparing strings.

To get the original string back, you can use the Core::ReverseUID() with the name_t returned by the Core::UID() function as the parameter.
